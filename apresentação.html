Pipeline RAG: Base de Conhecimento

Documentos acadêmicos reais transformados em contexto vetorial para respostas precisas.

6 Categorias de Fonte

Regras de matrícula, regulamento, FAQ, currículo, documentos e agendamento.

Chunking Otimizado

RecursiveCharacterTextSplitter com tokenizer cl100k_base — 500 tokens/chunk, 50 overlap.

Busca HNSW Sub-linear

Índice vetorial com cosine_ops (m=16, ef=64) direto no PostgreSQL — sem serviço externo.

Ingest Idempotente

DELETE + INSERT por source com audit trail — re-execução segura sem duplicatas.

Agente LangChain: Design de Produção

01

Provider-Agnóstico

Factory pattern suporta OpenAI, Gemini e OpenRouter — troca por env var, sem alterar código.

02

Stateless by Design

Agente reconstruído por request. Histórico (k=20) carregado do PostgreSQL via session header.

03

Guardrails de Execução

MAX_ITERATIONS=10 contra loops + timeout de 45s com fallback gracioso para a secretaria.

04

Tools Dinâmicas via MCP

langchain-mcp-adapters instancia client por sessão — tools carregadas em runtime do servidor.

MCP Tools: Confirmação e Observabilidade

Chain-of-Thought Capturado

Campo reasoning grava o raciocínio do agente antes de cada tool call.

Retry Discriminado

Erro 5xx → 1 retry imediato. Erro 4xx → sem retry (falha lógica). Status: success | error | retry_success.

Latência Medida
