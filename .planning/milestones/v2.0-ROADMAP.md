## Overview

Complete Flutter mobile/web application with role-based navigation (Client and Provider/Staff), consuming the FastAPI REST API built in M1. Includes OTP authentication, 6 client screens, 4+ staff screens, resource allocation, human intervention system, cross-platform responsiveness, glassmorphism visual identity ("Alpha Connect"), Docker Compose full-stack integration, and E2E test coverage.

## Milestone Summary

**Key Accomplishments:**
1. Flutter app boots with OTP auth, role-based navigation, secure JWT storage
2. 10+ screens (6 client + 4+ staff) consuming real REST API data
3. Alpha Connect glassmorphism visual identity with dark mode support
4. Docker Compose full-stack (5 services: fastapi, langchain, mcp, postgres, flutter-web)
5. Resource allocation with authorization uploads and human intervention system
6. 244 Flutter unit tests, E2E integration tests against live backend

**Key Decisions:**
- Riverpod + GoRouter for state/navigation (not Bloc)
- QueuedInterceptor for auth token serialization
- Glassmorphism UI with glassmorphism bottom nav (phone) / NavigationRail (tablet/desktop)
- Docker multi-stage build for Flutter web
- DEV_MASTER_OTP bypass for dev/testing
- Derived notifications (no backend endpoint — aggregates from documents + appointments)

**Issues Resolved:**
- Auth interceptor async void patterns (CR-01, CR-02)
- GlobalKey router crashes on auth transitions (CR-03)
- Debug logging leak to release builds
- API contract mismatches between Dart models and FastAPI responses
- CORS configuration for Flutter web

**Technical Debt Remaining:**
- 8 pre-existing test failures (staff_dashboard KPI card overflow — layout only)
- Push notifications (FCM) deferred
- Offline-first caching strategy deferred
- pg_cron scheduled cleanup deferred
