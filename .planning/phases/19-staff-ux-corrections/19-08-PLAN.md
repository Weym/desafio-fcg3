    CRITICAL: ChatSessionResponse uses `from_attributes = True` and is returned directly from ORM objects. Since `student_name` and `student_ra` are NOT columns on ChatSession, Pydantic will fail to populate them from the ORM object. You MUST override the serialization. Two approaches — use the `@model_validator(mode='before')` approach:

    Add a model validator to ChatSessionResponse:
    ```python
    from pydantic import model_validator

    class ChatSessionResponse(BaseModel):
        # ... existing fields ...
        student_name: str | None = None
        student_ra: str | None = None

        @model_validator(mode='before')
        @classmethod
        def extract_student_data(cls, data):
            """Extract student name and RA from loaded student relationship."""
            if hasattr(data, 'student') and data.student is not None:
                # ORM object — extract from relationship
                if not hasattr(data, 'student_name') or data.student_name is None:
                    # Use object_setattr trick or return dict
                    pass
            # For ORM objects, convert to dict first
            if hasattr(data, '__dict__'):
                d = {}
                for field in ['id', 'student_id', 'whatsapp_phone', 'status', 'name',
                              'verification_state', 'assigned_staff_id', 'escalated_at',
                              'started_at', 'ended_at', 'updated_at']:
                    d[field] = getattr(data, field, None)
                if hasattr(data, 'student') and data.student is not None:
                    d['student_name'] = data.student.name
                    d['student_ra'] = data.student.registration_number
                return d
            return data
    ```

    IMPORTANT: An alternative simpler approach — instead of the model_validator, manually build ChatSessionResponse in the router/service where sessions are returned. The `list_chat_sessions` endpoint at router.py line 72 currently does:
    ```python
    return ChatSessionListResponse(data=sessions, pagination=...)
    ```
    This passes raw ORM ChatSession objects into `data`. Change to build ChatSessionResponse explicitly:
    ```python
    session_responses = []
    for s in sessions:
        resp = ChatSessionResponse(
            id=s.id, student_id=s.student_id, whatsapp_phone=s.whatsapp_phone,
            status=s.status, name=s.name, verification_state=s.verification_state,
            assigned_staff_id=s.assigned_staff_id, escalated_at=s.escalated_at,
            started_at=s.started_at, ended_at=s.ended_at, updated_at=s.updated_at,
            student_name=s.student.name if s.student else None,
            student_ra=s.student.registration_number if s.student else None,
        )
        session_responses.append(resp)
    return ChatSessionListResponse(data=session_responses, pagination=...)
    ```

    Choose whichever approach is cleaner. The model_validator is DRY but complex; the explicit builder is simple and clear. **Prefer the explicit builder approach** in both `list_chat_sessions` and `list_intervention_sessions` router endpoints.
