Create an isolated WhatsApp client class encapsulating ALL Graph API details:

```python
import hashlib
import hmac
import httpx
import logging

logger = logging.getLogger(__name__)

class WhatsAppClient:
    """Encapsulates all WhatsApp Business Cloud API interactions."""
    
    def __init__(self, settings):
        self.phone_number_id = settings.WHATSAPP_PHONE_NUMBER_ID
        self.token = settings.WHATSAPP_TOKEN
        self.api_version = settings.WHATSAPP_API_VERSION
        self.base_url = f"https://graph.facebook.com/{self.api_version}/{self.phone_number_id}/messages"
        self._client = httpx.AsyncClient(
            timeout=httpx.Timeout(10.0, connect=5.0),
            headers={"Authorization": f"Bearer {self.token}", "Content-Type": "application/json"},
        )
    
    async def send_text_message(self, to: str, body: str) -> bool:
        """Send a text message via WhatsApp. Retries once on failure per D-07."""
        payload = {
            "messaging_product": "whatsapp",
            "to": to,
            "type": "text",
            "text": {"body": body},
        }
        for attempt in range(2):  # one retry per D-07
            try:
                response = await self._client.post(self.base_url, json=payload)
                if response.status_code == 200:
                    return True
                logger.warning(f"WhatsApp send attempt {attempt+1} failed: {response.status_code} {response.text}")
            except httpx.HTTPError as e:
                logger.warning(f"WhatsApp send attempt {attempt+1} error: {e}")
        logger.error(f"WhatsApp send failed after 2 attempts to {to}")
        return False  # Message still saved in chat_messages for audit per D-07
    
    async def close(self):
        await self._client.aclose()


def validate_signature(raw_body: bytes, signature_header: str, app_secret: str) -> bool:
    """Validate X-Hub-Signature-256 header. Must be called BEFORE any JSON parsing (CRITICAL-1)."""
    if not signature_header.startswith("sha256="):
        return False
    expected = "sha256=" + hmac.new(
        app_secret.encode(), raw_body, hashlib.sha256
    ).hexdigest()
    return hmac.compare_digest(signature_header, expected)
```

Implement the core webhook business logic:

- `lookup_student_by_phone(phone: str, db: AsyncSession) -> Optional[Student]` — direct string comparison per D-04 (phone stored in international format without +)
- `get_or_create_session(student_id: UUID, phone: str, db: AsyncSession) -> ChatSession` — reuse active session per D-10, create new if none active. When creating from a closed session's phone, new session starts `unverified` per D-13.
- `save_message(session_id: UUID, role: str, content: str, media_type: Optional[str], wamid: Optional[str], db: AsyncSession) -> Optional[ChatMessage]` — insert with deduplication by `whatsapp_message_id`. On IntegrityError (duplicate wamid per MODERATE-1), return None to signal dedup.
- `handle_verification_flow(session: ChatSession, message_text: str, phone: str, db: AsyncSession, wa_client: WhatsAppClient)` — implements the verification state machine per D-02:
  - State `unverified`: Send "Preciso verificar sua identidade. Qual seu email institucional?" → transition to `awaiting_email`
  - State `awaiting_email`: Parse email from message text. Look up student by email. If found and email matches session's student, call `POST /auth/request-code` internally (or reuse auth service directly). Send "Enviei um codigo para seu email. Informe o codigo de 6 digitos." → transition to `awaiting_code`. If email not found: "Nao encontrei cadastro com esse email. Verifique o email e tente novamente."
  - State `awaiting_code`: Parse 6-digit code from message text. Call `POST /auth/verify-code` internally. If valid: transition to `verified`, send "Identidade verificada! Ola, {name}. Como posso ajudar?". If invalid (attempts < 3): "Codigo invalido. Tente novamente. ({N} tentativa(s) restante(s))". If max attempts: "Codigo invalido. Limite atingido. Enviei um novo codigo para seu email."
  - State `verified`: Message passes through to agent processing (handled in Plan 02).
- `get_media_response(media_type: str) -> str` — returns exact Portuguese response from `docs/chatbot.md` media type table. Hardcoded, no LLM (per D-02 specifics section).

CRITICAL: Verification flow is pre-agent per D-02. Unverified students NEVER reach LangChain agent.
CRITICAL: For unknown phone numbers (no student found), send "Nao encontrei cadastro para este numero. Procure a secretaria para cadastro." and do NOT create a chat_session per D-03.

**3. Create Webhook Router (backend/src/features/webhook/router.py):**

```python
@router.get("/webhook/whatsapp")
async def webhook_challenge(
    hub_mode: str = Query(None, alias="hub.mode"),
    hub_verify_token: str = Query(None, alias="hub.verify_token"),
    hub_challenge: str = Query(None, alias="hub.challenge"),
):
    """Webhook verification challenge per WH-01."""
    if hub_mode == "subscribe" and hub_verify_token == settings.WHATSAPP_VERIFY_TOKEN:
        return PlainTextResponse(content=hub_challenge)
    raise HTTPException(status_code=403, detail="Verification failed")

@router.post("/webhook/whatsapp")
async def whatsapp_webhook(request: Request):
    """Receive WhatsApp messages per WH-02, WH-03, WH-04."""
    # CRITICAL-1: Read raw body FIRST before any parsing
    raw_body = await request.body()
    
    # Validate HMAC-SHA256 signature
    signature = request.headers.get("X-Hub-Signature-256", "")
    if not validate_signature(raw_body, signature, settings.WHATSAPP_APP_SECRET):
        raise HTTPException(status_code=403, detail="Invalid signature")
    
    # Parse payload manually (not via Pydantic body param)
    payload = WhatsAppWebhookPayload.model_validate_json(raw_body)
    
    # Filter out status updates (delivery receipts) — MODERATE-1
    for entry in payload.entry:
        for change in entry.changes:
            if not change.value.messages:
                continue  # skip status updates
            for message in change.value.messages:
                phone = message.from_
                wamid = message.id
                
                # Open own DB session (CRITICAL-4 — not request-scoped)
                async with async_session_maker() as db:
                    # Phone lookup per D-04
                    student = await webhook_service.lookup_student_by_phone(phone, db)
                    if student is None:
                        # D-03: Unknown phone, send rejection, no session
                        await wa_client.send_text_message(phone, "Nao encontrei cadastro para este numero. Procure a secretaria para cadastro.")
                        continue
                    
                    session = await webhook_service.get_or_create_session(student.id, phone, db)
                    
                    # Handle media messages immediately (no agent) per docs/chatbot.md
                    if message.type != "text":
                        media_response = webhook_service.get_media_response(message.type)
                        await webhook_service.save_message(session.id, "user", f"[{message.type}]", message.type, wamid, db)
                        await webhook_service.save_message(session.id, "assistant", media_response, None, None, db)
                        await wa_client.send_text_message(phone, media_response)
                        continue
                    
                    # Save user message with dedup
                    msg = await webhook_service.save_message(session.id, "user", message.text.body, None, wamid, db)
                    if msg is None:
                        continue  # duplicate wamid, skip
                    
                    # Verification state machine per D-02
                    if session.verification_state != "verified":
                        await webhook_service.handle_verification_flow(session, message.text.body, phone, db, wa_client)
                        continue
                    
                    # Verified: dispatch to background task (Plan 02 handles this)
                    task = asyncio.create_task(
                        process_verified_message(session.id, message.text.body, phone)
                    )
                    task.add_done_callback(_handle_task_result)  # CRITICAL-3
    
    return Response(status_code=200)
```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| WhatsApp → FastAPI webhook | External untrusted HTTP from Meta servers; validated by HMAC-SHA256 |
| Webhook → DB | User input (message content) stored in database |
| Webhook → WhatsApp Graph API | Outbound HTTP with Bearer token |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01 | Spoofing | POST /webhook/whatsapp | mitigate | HMAC-SHA256 validation using raw body bytes before any parsing (CRITICAL-1). Reject with 403 on mismatch. |
| T-06-02 | Tampering | WhatsApp message payload | mitigate | Raw body read first, then parsed; no Pydantic body parameter. Signature covers entire payload. |
| T-06-03 | Repudiation | Message processing | mitigate | All messages saved to chat_messages with whatsapp_message_id for audit trail. |
| T-06-04 | Information Disclosure | WhatsApp tokens | mitigate | All tokens in env vars only, never in source code (per AGENTS.md constraint). |
| T-06-05 | Denial of Service | Duplicate webhook deliveries | mitigate | Deduplication by whatsapp_message_id unique constraint; IntegrityError caught gracefully. |
| T-06-06 | Elevation of Privilege | Unverified user accessing agent | mitigate | Verification state machine gates agent access; unverified sessions never reach LangChain (D-02). |
| T-06-07 | Spoofing | Webhook challenge | mitigate | Verify token compared against env var; 403 on mismatch. |
</threat_model>
