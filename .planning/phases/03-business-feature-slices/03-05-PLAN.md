## Key Decisions
- **D-07:** CRA calculation at service level in Python, not SQL. More testable and readable.
- **D-08:** Include only grades with `final_grade IS NOT NULL`. In-progress and locked courses excluded. If no completed grades → CRA = 0.0.

## API Endpoints (from docs/api.md)
- `GET /students/{id}/grades` — student/staff, MCP-accessible, filter by semester_year
- `GET /students/{id}/transcript` — student/staff, MCP-accessible, full history
- `PUT /grades/{id}` — staff only, post/update grades (N1, N2)

## Database Tables
- `grades`: id, student_id, course_id, enrollment_course_id, semester_year, grade_1, grade_2, grade_final, status, created_at, updated_at
- `courses`: credits (needed for CRA weighting)
- `enrollment_courses`: enrollment_id, course_id, status

## CRA Formula
CRA = Σ(grade_final × credits) / Σ(credits)
Where: only grades with final_grade IS NOT NULL are included.
Division by zero guard: if sum of credits = 0 → return 0.0.
</context>
