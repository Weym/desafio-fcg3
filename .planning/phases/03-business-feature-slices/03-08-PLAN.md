## API Endpoint (from docs/api.md)
- `GET /staff/dashboard` — staff only. Returns KPI aggregation.

## Response Shape (from docs/api.md)
```json
{
  "total_students": 500,
  "active_enrollments": 450,
  "pending_documents": 23,
  "upcoming_appointments": 8,
  "active_chat_sessions": 3,
  "enrollment_period": {
    "name": "2025.1",
    "is_active": true,
    "days_remaining": 12
  }
}
```

## Tables Queried
- `students` — count where status='active'
- `enrollments` — count where status='confirmed'
- `documents` — count where status IN ('requested', 'processing')
- `appointments` — count where status='scheduled' AND date >= today
- `chat_sessions` — count where status='active'
- `enrollment_periods` — get active period, calculate days remaining
</context>

    Create `DashboardService`:

    1. `get_dashboard(db)` — STAFF-01:
       Execute 6 efficient count queries (can be parallelized or combined):

       a) `total_students`: `SELECT COUNT(*) FROM students WHERE status = 'active'`
       b) `active_enrollments`: `SELECT COUNT(*) FROM enrollments WHERE status = 'confirmed'`
       c) `pending_documents`: `SELECT COUNT(*) FROM documents WHERE status IN ('requested', 'processing')`
       d) `upcoming_appointments`: `SELECT COUNT(*) FROM appointments WHERE status = 'scheduled' AND slot_id IN (SELECT id FROM scheduling_slots WHERE date >= CURRENT_DATE)`
       e) `active_chat_sessions`: `SELECT COUNT(*) FROM chat_sessions WHERE status = 'active'`
       f) `enrollment_period`: Get active enrollment period. If found, calculate `days_remaining = (end_date - today).days`. If no active period, return None.

       Use SQLAlchemy `func.count()` with the ORM models. For efficiency, consider executing counts in parallel using `asyncio.gather()` on separate queries, or a single query with multiple subselects.

       Return DashboardResponse.

    **backend/src/features/staff/controllers.py:**
    1. `GET /staff/dashboard` — `require_staff` dependency. Calls DashboardService.get_dashboard(). Returns DashboardResponse.
