## API Endpoints (from docs/api.md)
- `POST /documents` — student, MCP-accessible. Creates document request.
- `GET /documents` — authenticated, filters: student_id, type, status
- `GET /documents/{id}` — MCP-accessible. Detail with file_url when ready.
- `PUT /documents/{id}/status` — staff only. Updates status and attaches file_url.

## Database Table
- `documents`: id, student_id, type (transcript/enrollment_proof/declaration/certificate), status (requested/processing/ready/delivered), file_url, requested_at, completed_at
- **Schema mismatch (SM-02 from 03-RESEARCH.md):** `docs/api.md` POST /documents request includes `notes` field but `docs/database.md` `documents` table has no `notes` column. This plan must add an Alembic migration to create `notes TEXT NULLABLE` on the `documents` table.

## Document Status Lifecycle
- `requested` → `processing` → `ready` → `delivered`
