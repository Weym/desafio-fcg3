    **backend/src/features/courses/schemas.py:**
    - `CourseListItem(id, code, name, credits, workload_hours)` — for list
    - `PrerequisiteItem(id, code, name)` — direct prerequisite
    - `CourseDetail(id, code, name, credits, workload_hours, description, prerequisites: list[PrerequisiteItem])` — for detail
    - `PrerequisiteTreeNode(id, code, name, prerequisites: list[PrerequisiteTreeNode])` — recursive tree node. Use Pydantic's `model_rebuild()` for self-referential model.
    - `CurriculumCourseItem(id, code, name, credits, is_required)` — course within a semester
    - `SemesterGroup(semester: int, courses: list[CurriculumCourseItem])` — semester grouping
    - `CurriculumResponse(id, name, year, semesters: list[SemesterGroup])` — full curriculum

    **backend/src/features/courses/services.py:**
    Create `CourseService`:

    1. `list_courses(db, params, search?, semester?)` — list courses with ILIKE search on name/code. If `semester` filter provided, join curriculum_courses and filter. Use BaseService.list pattern.

    2. `get_course_detail(db, course_id)` — get course by ID with direct prerequisites (single JOIN on prerequisites table). Returns CourseDetail with prerequisites list.

    3. `get_prerequisite_tree(db, course_id)` — COURSE-03: Execute a **recursive CTE** query against the `prerequisites` table to build the full tree. SQL pattern:
    ```sql
    WITH RECURSIVE prereq_tree AS (
        SELECT p.prerequisite_id, c.code, c.name, p.course_id, 1 as depth
        FROM prerequisites p
        JOIN courses c ON c.id = p.prerequisite_id
        WHERE p.course_id = :course_id
        UNION ALL
        SELECT p.prerequisite_id, c.code, c.name, p.course_id, pt.depth + 1
        FROM prerequisites p
        JOIN courses c ON c.id = p.prerequisite_id
        JOIN prereq_tree pt ON p.course_id = pt.prerequisite_id
    )
    SELECT * FROM prereq_tree
    ```
    Use `text()` for the raw SQL or SQLAlchemy Core CTE construct. Build the tree structure in Python from the flat CTE results (group by parent). Add depth limit (max 10) to prevent infinite recursion from circular prereqs.

    4. `get_active_curriculum(db)` — CURR-01: Query curriculum where `is_active = True`. Join curriculum_courses and courses. Group by semester. Return CurriculumResponse.

    5. `get_curriculum_by_id(db, curriculum_id)` — CURR-02: Same as active but by specific ID. Raises NotFoundException with resource_name="curriculo" if not found.

    Create `CurriculumService` (or keep in same CourseService — agent discretion).
