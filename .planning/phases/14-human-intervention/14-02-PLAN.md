## Trust Boundaries

| Boundary | Description |
| --- | --- |
| Staff app → Backend API | JWT-authenticated calls to assign/reply/resolve endpoints |
| Staff input → WhatsApp message | Staff-typed content sent to student — must be non-empty |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
| --- | --- | --- | --- | --- |
| T-14-07 | Spoofing | Staff identity on reply | mitigate | JWT auth enforced by DioClient interceptor; backend validates staff role + ownership |
| T-14-08 | Tampering | Empty reply content | mitigate | TextField validates non-empty before enabling send; backend enforces min_length=1 |
| T-14-09 | Information Disclosure | Student data in intervention list | accept | Staff role inherently has access to student data per system design — role guard at route level prevents student access |
| T-14-10 | Denial of Service | Rapid-fire reply spam | accept | WhatsApp API rate limiting + backend per-session lock provides natural throttling |

</threat_model>
