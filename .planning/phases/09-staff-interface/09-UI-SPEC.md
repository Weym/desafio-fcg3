# Phase 9 — UI Design Contract

> Visual and interaction contract for the Staff Interface phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | Material 3 / Flutter ThemeData |
| Preset | `ColorScheme.fromSeed(seedColor: Color(0xFF1565C0))` |
| Component library | Flutter Material (built-in) |
| Icon library | Material Icons (outlined variant default, filled for active states) |
| Font | System default (Roboto on Android, SF Pro on iOS) |

---

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, chip internal padding vertical, timestamp spacing |
| sm | 8px | Compact element spacing: between filter chips, between chip row and content, card bottom margins |
| md | 12px | Card internal padding, icon-to-text gaps within cards |
| lg | 16px | Default body padding (horizontal), ListView item padding, section spacing |
| xl | 24px | Bottom sheet internal padding, section headers, between dashboard KPI grid rows |
| 2xl | 32px | Not used in this phase |
| 3xl | 48px | Minimum touch targets (icon containers: 48x48) |

Exceptions:
- KPI card icon containers: 44x44 (matches established client pattern for icon containers in cards)
- Touch targets on action buttons: minimum 48dp per Material accessibility guidelines
- Bottom sheet top radius: 16px (established in `document_request_sheet.dart`)
- 12px (md token) included per Material Design card padding convention and Phase 8 precedent (`_DocumentCard` uses `EdgeInsets.all(12)`)

## Typography

| Role | TextStyle | Size | Weight | Line Height | Usage |
|------|-----------|------|--------|-------------|-------|
| Display | `headlineLarge` | 28px | 700 (bold) | 1.2 | KPI numbers on dashboard cards |
| Heading | `titleLarge` | 20px | 500 (medium) | 1.3 | Bottom sheet titles, detail screen titles |
| Subheading | `titleSmall` | 14px | 600 (semibold) | 1.4 | Card titles, section labels |
| Body | `bodyMedium` | 14px | 400 (regular) | 1.5 | Card subtitles, descriptions, list content |

Note: Use `Theme.of(context).textTheme.*` exclusively — never hardcode font sizes inline except for status chips (12px as established in client documents pattern). Weights follow Material 3 TextTheme role defaults — not independently chosen; they are documented here for executor reference only.

## Component Patterns

### KPI Dashboard Grid (D-01, D-02)

```
┌──────────────────────────────────────┐
│ [Enrollment Period Banner]            │  ← Card with primary accent
│  Nome do Período  [Badge: Ativo]     │
│  "X dias restantes"                  │
└──────────────────────────────────────┘

┌────────────┐  ┌────────────┐
│ 💎 icon    │  │ 💎 icon    │
│   42       │  │   18       │
│ Alunos     │  │ Matrículas │
└────────────┘  └────────────┘
┌────────────┐  ┌────────────┐
│ 💎 icon    │  │ 💎 icon    │
│    7       │  │    3       │
│ Docs Pend. │  │ Agendamen. │
└────────────┘  └────────────┘
┌────────────┐  ┌────────────┐
│ 💎 icon    │  │            │
│    5       │  │ (empty)    │
│ Chats Ativ │  │            │
└────────────┘  └────────────┘
```

- Grid: `GridView.count(crossAxisCount: 2, crossAxisSpacing: 12, mainAxisSpacing: 12, childAspectRatio: 1.4)`
- Card border radius: 12px
- Card elevation: 1
- Icon container: 44x44, borderRadius: 10, background: `iconColor.withOpacity(0.12)`
- Number: `headlineLarge` (28px, bold)
- Label: `bodySmall` (12px, `onSurfaceVariant` color)
- Enrollment banner: full-width Card with `primaryContainer` background, `primary` text

### Filter Chips Row (D-06, D-09, D-14)

- `SingleChildScrollView(scrollDirection: Axis.horizontal)`
- Padding: `EdgeInsets.symmetric(horizontal: 16, vertical: 8)`
- Chip spacing: `SizedBox(width: 8)`
- Toggle behavior: tapping active chip calls `setFilter(null)` (resets to "Todos")
- Use Material `FilterChip` widget with `selected` state

### List Cards (D-06, D-11, D-14)

- Card margin: `EdgeInsets.only(bottom: 8)`
- Card border radius: 12px
- Card internal padding: 12px
- Structure: `Row` with leading icon container (44x44) → SizedBox(12) → Expanded column (title + subtitle) → trailing status chip
- Title: `titleSmall` with `fontWeight: FontWeight.w600`
- Subtitle: `bodySmall` with `onSurfaceVariant` color
- Status chip: Container with 12px borderRadius, 8px horizontal / 4px vertical padding

### Bottom Sheets (D-08, D-15, D-17)

- Shape: `RoundedRectangleBorder(borderRadius: BorderRadius.vertical(top: Radius.circular(16)))`
- `isScrollControlled: true`
- Padding: `EdgeInsets.fromLTRB(24, 24, 24, MediaQuery.of(context).viewInsets.bottom + 24)`
- Title: `titleLarge` (20px)
- Field spacing: `SizedBox(height: 16)`
- Title-to-first-field gap: `SizedBox(height: 24)`
- Submit button: full-width `ElevatedButton` with `minimumSize: Size(double.infinity, 48)`
- Loading state: replace button text with 20x20 `CircularProgressIndicator(strokeWidth: 2)`

### Confirmation Dialog (D-07 — destructive actions)

- Use `showDialog` with `AlertDialog`
- Title: action-specific (e.g., "Cancelar Agendamento")
- Content: confirmation question with consequence description
- Actions: "Voltar" (TextButton, no color) + action-specific button (TextButton, `colorScheme.error` for destructive)
- No dismiss on barrier tap for destructive actions (`barrierDismissible: false`)

### Appointment Detail Screen (D-07)

- Pushed as a new route (not bottom sheet)
- AppBar with back button, title "Detalhes do Agendamento"
- Body: `SingleChildScrollView` with `padding: EdgeInsets.all(16)`
- Info rows: `ListTile`-style with label (bodySmall) + value (bodyMedium)
- Action buttons at bottom: Row with "Confirmar Agendamento" (ElevatedButton, primary) + "Cancelar Agendamento" (OutlinedButton, error color)
- Buttons have minimum height 48px

### AI Data Tabs (D-10)

- `TabBar` in AppBar bottom (same as `ClientChatDetailScreen`)
- 2 tabs: "Sessoes" + "Estatisticas"
- `ConsumerStatefulWidget` with `SingleTickerProviderStateMixin` for TabController

### FAB (D-08, D-17)

- `FloatingActionButton.extended` with icon + label
- Schedule screen: `icon: Icons.add`, `label: "Novo Slot"`
- Documents screen: `icon: Icons.add`, `label: "Enviar"`

- Primary: `Center(child: CircularProgressIndicator())` — used for initial data fetch

## Interaction Contracts

### Navigation (D-03)

- Tap on KPI card navigates to corresponding tab via `context.go(RoutePaths.staffX)`
- KPI "Docs Pendentes" → staffDocuments
- KPI "Agendamentos" → staffSchedule
- KPI "Chats Ativos" → staffAI
- KPI "Alunos" / "Matriculas" → no navigation (informational only)

### Pull-to-Refresh (D-04, D-19)

- Wrap scrollable content in `RefreshIndicator`
- On refresh: `ref.invalidate(provider)` + `await ref.read(provider.future)`
- Dashboard refreshes ALL KPI providers simultaneously

### Filter Chips (D-09)

- Toggle behavior: if chip is already selected, tapping it resets filter to null ("Todos")
- Implementation: `onSelected: (_) => ref.read(filterProvider.notifier).setFilter(filter == value ? null : value)`

### FAB Actions (D-08, D-17)

- Opens bottom sheet with form
- Sheet is scroll-controlled to respect keyboard insets
- Form validation before submit
- Optimistic: invalidate provider after success, show SnackBar, pop sheet

### Detail Navigation (D-07, D-13)

- Appointment detail: pushed sub-route `/staff/schedule/:appointmentId`
- AI session detail: pushed sub-route `/staff/ai/:sessionId` — reuses `ClientChatDetailScreen` layout pattern
- Both use GoRouter `context.push()` (push, not go — preserves nav stack)

### Autocomplete (D-17 — student search)

- Use Flutter's built-in `Autocomplete<StudentModel>` widget
- Debounce: 300ms before triggering API call
- Minimum 2 characters before search fires
- Display: student name + email in suggestion tile
- On select: store `student_id` in form state

### File Upload (D-15, D-16, D-17)

- Use `file_picker` package (Flutter standard, no native deps beyond what Flutter provides)
- File picker button appears conditionally when status = "ready" (D-15)
- Upload flow: pick file → show filename + progress indicator → call `POST /documents/upload` → receive URL → store in form state
- File types allowed: PDF, PNG, JPG (validated in picker config)
- Max file size: 10MB (validate before upload, show error if exceeded)

### Pagination (Agent's Discretion — resolved)

- Use simple list with `RefreshIndicator` (pull-to-refresh) for initial implementation
- No infinite scroll — staff datasets expected to be < 100 items per screen
- If list exceeds viewport, natural `ListView.builder` scrolling handles it
- Future optimization (Phase 10): add pagination if performance requires it
