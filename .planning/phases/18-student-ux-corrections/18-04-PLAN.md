**STUX-11, STUX-12, STUX-13 — Notification read/unread state:**

Currently notifications are "derived" from documents and appointments (no backend notification entity). The read/unread state must be managed client-side (persisted in local state since there's no notification table).

In `notification_provider.dart`, add:

1. **A Riverpod StateNotifier for read notification IDs** (persisted in memory for the session; could be extended to SharedPreferences later):

```dart
@riverpod
class ReadNotificationIds extends _$ReadNotificationIds {
  @override
  Set<String> build() => {};
  
  void markAsRead(String id) {
    state = {...state, id};
  }
  
  void markAllAsRead(List<String> ids) {
    state = {...state, ...ids};
  }
}
```

2. **Add `isRead` computed property to `DerivedNotification`:**
Since the DerivedNotification is a plain class, we don't modify it. Instead, in the screen, we'll check against the readNotificationIds set.

3. **Add a notification filter state provider:**
```dart
enum NotificationFilter { all, unread, read }

@riverpod
class NotificationFilterNotifier extends _$NotificationFilterNotifier {
  @override
  NotificationFilter build() => NotificationFilter.all;
  
  void setFilter(NotificationFilter filter) => state = filter;
}
```

3. **Filter notifications based on read state:**
```dart
final filtered = switch (filter) {
  NotificationFilter.all => notifications,
  NotificationFilter.unread => notifications.where((n) => !readIds.contains(n.id)).toList(),
  NotificationFilter.read => notifications.where((n) => readIds.contains(n.id)).toList(),
};
```

4. **Add "Visualizar todos" button** in the header area (next to the count text):
```dart
Row(
```
