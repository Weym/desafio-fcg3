## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
| --- | --- | --- | --- | --- |
| T-13-10 | Spoofing | Booking as another student | mitigate | Backend extracts student_id from JWT token (UserContext) â€” never from request body. Client cannot spoof identity. |
| T-13-11 | Tampering | Uploading malicious file | mitigate | Client-side: restrict to pdf/jpg/png extensions via FilePicker. Server-side: validates content_type allowlist + 5MB limit. File stored with UUID prefix (no path traversal). |
| T-13-12 | Information Disclosure | Viewing other students' appointments | mitigate | Backend auto-filters appointments to authenticated student_id (IDOR-safe in service layer). |
| T-13-13 | Denial of Service | Rapid slot booking | accept | SELECT FOR UPDATE prevents double-booking; worst case is 409 error to second student. Acceptable for MVP. |
| T-13-14 | Elevation of Privilege | Student accessing staff-only resource data | mitigate | Backend GET /resources for students auto-filters to is_available=true only; soft-deleted resources invisible. |

</threat_model>
