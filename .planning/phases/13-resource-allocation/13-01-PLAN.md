<threat_model>

## Trust Boundaries

| Boundary | Description |
| --- | --- |
| Client → /resources | Untrusted input for resource CRUD (staff) and listing (student) |
| Client → /appointments/{id}/authorization | File upload from untrusted client |
| Student → Appointment ownership | Student must only access own appointments |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
| --- | --- | --- | --- | --- |
| T-13-01 | Elevation of Privilege | POST/PUT/DELETE /resources | mitigate | `require_staff(user)` check on every mutating endpoint — students get 403 |
| T-13-02 | Tampering | File upload content type | mitigate | Validate content_type server-side against allowlist ['application/pdf', 'image/jpeg', 'image/png']; reject others with 400 |
| T-13-03 | Denial of Service | Large file upload | mitigate | Enforce 5MB limit via `UploadFile.size` check before reading full body; return 400 if exceeded |
| T-13-04 | Information Disclosure | GET /resources by student | mitigate | Students auto-filtered to `is_available=True` only; staff sees all including soft-deleted |
| T-13-05 | Spoofing | Upload to another student's appointment | mitigate | IDOR check: verify `appointment.student_id == user.id` before allowing upload (staff can upload for any) |
| T-13-06 | Tampering | Path traversal in filename | mitigate | Use UUID prefix + sanitized filename; never use raw user filename in path construction |

</threat_model>
