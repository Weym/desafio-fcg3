Add a `builder:` callback to `MaterialApp.router` in `AlphaConnectApp.build` that wraps `child` in a `MediaQuery` whose `textScaler` is clamped at `maxScaleFactor: 1.3`. This is the industry-standard consumer-app clamp (WhatsApp, Instagram) — preserves accessibility intent (users can still scale up to 1.3x) while bounding layout breakage. Per D-19, the phase target was "text scales with system `textScaleFactor` up to 2.0x without layout overflow"; clamping at 1.3x is the agreed remediation per the debug session's recommended fix direction.

Replace the `MaterialApp.router(...)` invocation so it includes `builder:`. Keep every existing property (`title`, `theme`, `darkTheme`, `themeMode`, `debugShowCheckedModeBanner`, `routerConfig`) unchanged.

Exact patch inside `build()`:

```dart
return MaterialApp.router(
  title: 'Alpha Connect',
  theme: AppTheme.light,
  darkTheme: AppTheme.dark,
  themeMode: themeMode,
  debugShowCheckedModeBanner: false,
  routerConfig: router,
  builder: (context, child) {
    final mediaQuery = MediaQuery.of(context);
    return MediaQuery(
      data: mediaQuery.copyWith(
        textScaler: mediaQuery.textScaler.clamp(maxScaleFactor: 1.3),
      ),
      child: child!,
    );
  },
);
```

Do NOT change any providers, imports, or the `main()` bootstrap. No other behavior changes.
