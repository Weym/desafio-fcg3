## Spacing Scale

Declared values (multiples of 4dp). Defined as constants in `lib/core/theme/app_spacing.dart`.

| Token | Value (dp) | Dart Constant | Usage |
|-------|------------|---------------|-------|
| xs | 4 | `AppSpacing.xs` | Icon gaps, inline padding, tight list separators |
| sm | 8 | `AppSpacing.sm` | Compact element spacing, chip padding |
| md | 16 | `AppSpacing.md` | Default element spacing, card content padding |
| lg | 24 | `AppSpacing.lg` | Section padding, card-to-card gap |
| xl | 32 | `AppSpacing.xl` | Layout column gaps, major section breaks |

Source: D-23 from 10-CONTEXT.md.

Exceptions:
- Touch targets: minimum 48dp height/width per D-19 (WCAG AA)
- NavigationRail compact width: 72dp (D-01)
- NavigationRail extended width: 180dp (D-01)
- Max-width content container: 720dp (D-04)

## Responsive Breakpoints

## Typography

Material 3 `TextTheme` used directly. Responsive scaling applied on desktop (>= 1024dp).

### Phone / Tablet (standard Material 3 sizes)

| Role | TextStyle | Size | Weight | Line Height |
|------|-----------|------|--------|-------------|
| Body | `bodyMedium` | 14sp | 400 (regular) | 1.43 (20sp) |
| Body Large | `bodyLarge` | 16sp | 400 (regular) | 1.5 (24sp) |
| Label | `labelLarge` | 14sp | 500 (medium) | 1.43 (20sp) |
| Title | `titleMedium` | 16sp | 500 (medium) | 1.5 (24sp) |
| Headline | `headlineMedium` | 28sp | 400 (regular) | 1.29 (36sp) |
| Display | `displaySmall` | 36sp | 400 (regular) | 1.22 (44sp) |

### Desktop Scaling (>= 1024dp)

Headings and display styles scale ~20% larger. Body text unchanged; line-height increases to 1.6 for readability.

| Role | TextStyle | Desktop Size | Desktop Line Height |
|------|-----------|-------------|---------------------|
| Body | `bodyMedium` | 14sp (unchanged) | 1.6 |
| Body Large | `bodyLarge` | 16sp (unchanged) | 1.6 |
| Headline | `headlineMedium` | 34sp (+20%) | 1.29 |
| Display | `displaySmall` | 43sp (+20%) | 1.22 |

Font weights used: **400 (regular)** and **500 (medium)** only.

Source: D-24, D-25 from 10-CONTEXT.md.

### Text Scaling Accessibility

Text must scale with system `textScaleFactor` up to 2.0x without layout overflow. Source: D-19.

## Color

Material 3 `ColorScheme.fromSeed` generates both light and dark palettes from a single seed.

### Seed Color

```dart
seedColor: Color(0xFF1565C0) // Blue 800
```

### Light Theme Palette (existing, confirmed)

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#FFFFFF` / `surface` | Page backgrounds, scaffold |
| Secondary (30%) | `#FAFAFA` / `surfaceContainerLow` | Cards, tiles, elevated surfaces |
| Accent (10%) | `#1565C0` / `primary` | FABs, active nav icons, primary buttons, switches ON state |
| Destructive | `#D32F2F` / `error` | Cancel/delete actions, error text, error icons |
| On-surface text | `#1C1B1F` | Primary text, icons on surfaces |
| Outline | `#79747E` | Borders, dividers, disabled states |

### Dark Theme Palette (new — generated from same seed)

| Role | Value (generated) | Usage |
|------|-------------------|-------|
| Dominant (60%) | `ColorScheme.fromSeed(..., brightness: Brightness.dark).surface` | Dark scaffold background |
| Secondary (30%) | `surfaceContainerLow` (dark variant) | Cards, elevated surfaces in dark |
| Accent (10%) | `primary` (dark variant — lighter blue) | Same elements as light theme |
| Destructive | `error` (dark variant) | Same elements as light theme |

Source: D-16 from 10-CONTEXT.md.

### Accent Reserved For (explicit list)

- Primary action buttons (ElevatedButton with `primary` background)
- FloatingActionButton
- Active BottomNavigationBar / NavigationRail icon
- Active toggle/switch ON state
- Text links / interactive text
- LinearProgressIndicator (refresh indicator)
- Filter chip selected state

### Dark Mode Behavior

| Property | Value |
|----------|-------|
| Default | `ThemeMode.system` (follows OS) |
| Override | User manual toggle persisted in `SharedPreferences` |
| Toggle location | AppBar action icon (agent's discretion for exact position) |
| Persistence key | `theme_mode` in `SharedPreferences` |

Source: D-16, D-17, D-18 from 10-CONTEXT.md.

### WCAG AA Contrast Requirements

| Context | Minimum Ratio |
|---------|---------------|
| Text on surface | 4.5:1 |
| UI components (icons, borders) | 3:1 |
| Both light AND dark themes | Must independently pass |

Source: D-19 from 10-CONTEXT.md.

## Shared Widgets Contract

All screens must use these shared widgets for consistency. Located in `lib/shared/widgets/` (or `lib/core/widgets/`).

### Loading States

| Widget | Behavior | Source |
|--------|----------|--------|
| `AppSkeletonList` | Configurable list skeleton (N items of shimmer placeholders matching target layout shape) | D-08, D-11 |
| `AppSkeletonCard` | Configurable card skeleton (shimmer placeholder matching card dimensions) | D-08, D-11 |
| Skeleton → Content transition | Instantaneous (no fade animation) | D-10 |

Skeleton placeholders must mirror the actual screen layout (card shapes, list item heights) — not generic rectangles. Source: CONTEXT.md specifics section.

### Refresh States

| Trigger | Visual Feedback | Data Behavior |
|---------|-----------------|---------------|
| First load (no cache) | Skeleton/shimmer screen | Fetch from API |
| Pull-to-refresh (cached data visible) | `LinearProgressIndicator` at top | Existing data stays visible during fetch |
| Tab prefetch (background) | No visible indicator | Silent fetch in background |

Source: D-08, D-09, D-14 from 10-CONTEXT.md.

### Empty State

| Widget | Properties |
|--------|------------|
| `AppEmptyState` | `icon` (IconData) + `message` (String) + optional `actionLabel` (String) + optional `onAction` (VoidCallback) |

### Error State

| Widget | Properties |
|--------|------------|
| `AppErrorState` | `icon` (IconData, default: `Icons.error_outline`) + `message` (String) + `retryLabel` (String, default: "Tentar novamente") + `onRetry` (VoidCallback) |

### Offline Banner

| Widget | Behavior |
|--------|----------|
| `AppOfflineBanner` | Thin persistent strip at top of scaffold. Text: "Sem conexao". Action buttons disabled across app. Shows when connectivity lost; hides when restored. |

Source: D-15 from 10-CONTEXT.md.

## Data Cache & Synchronization Contract

| Property | Value | Source |
|----------|-------|--------|
| Cache strategy | Riverpod `keepAlive` + TTL invalidation | D-12 |
| TTL duration | 5 minutes (uniform across all providers) | D-12 |
| TTL behavior | After expiry, next access triggers refetch; during window, serves from memory | D-13 |
