  <action>
Create `ai_service/entrypoint.sh` that:
1. Runs RAG ingest (non-blocking on failure — the service should start even if ingest fails)
2. Then starts the AI service via the existing command

```bash
#!/bin/bash
set -eu

echo "[entrypoint] Running RAG knowledge base ingest..."

# Run ingest — failure is non-fatal (service starts regardless)
python -m ai_service.ingest --source /app/ai_service/knowledge 2>&1 || {
    echo "[entrypoint] WARNING: RAG ingest failed (database may not be ready or missing tables). Service will start anyway."
}

echo "[entrypoint] Starting AI service..."
exec python -m ai_service.main
```

Make the file executable (chmod +x).
  </action>
  <acceptance_criteria>
    - File ai_service/entrypoint.sh exists
    - File contains "python -m ai_service.ingest"
    - File contains "exec python -m ai_service.main"
    - File starts with "#!/bin/bash"
    - Ingest failure does not prevent service start (|| pattern present)
  </acceptance_criteria>
  <verify>
    <automated>test -f ai_service/entrypoint.sh && grep -q "python -m ai_service.ingest" ai_service/entrypoint.sh && grep -q "exec python -m ai_service.main" ai_service/entrypoint.sh && echo "PASS" || echo "FAIL"</automated>
  </verify>
  <done>Entrypoint script runs ingest then starts service, with graceful failure handling for ingest.</done>

  <action>
**1. Update `ai_service/Dockerfile`:**

Add the entrypoint script to the image. After the COPY step that copies the app code, add:
```dockerfile
COPY entrypoint.sh /app/entrypoint.sh
RUN chmod +x /app/entrypoint.sh
```

Do NOT change the CMD — the docker-compose command override will handle execution.

**2. Update `docker-compose.yml`:**

In the `langchain-service` section, change the `command` from:
```yaml
command: python -m ai_service.main
```
to:
```yaml
command: bash /app/entrypoint.sh
```

This ensures the entrypoint runs ingest first, then starts the service. The `depends_on: postgres: condition: service_healthy` already ensures the database is ready before the langchain-service starts.
  </action>

    - ai_service/Dockerfile contains "chmod +x"
    - docker-compose.yml langchain-service command references "entrypoint.sh"
    - docker-compose.yml still has depends_on postgres with service_healthy condition
  </acceptance_criteria>
  <verify>

  </verify>
