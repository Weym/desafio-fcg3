Add a `log_rag_invocation` function that inserts into `rag_logs` after each RAG query. The RAG tool must accept a `session_id` parameter (passed at creation time, not exposed to the LLM) and log the results.

Modify `create_rag_tool` to accept an optional `session_id: str | None = None` parameter. After executing the query, insert a log row with:
- `query`: the normalized search query
- `chunks_retrieved`: JSON array of `{"source", "category", "score", "chunk_index"}` for each retrieved chunk
- `threshold_met`: True if at least one chunk was returned (score >= threshold)

The logging must NOT block or fail the RAG response — wrap in try/except and log warnings on failure.

```python
def _log_rag_invocation(
    db_pool,
    session_id: str | None,
    query: str,
    chunks_list: list[dict],
    threshold_met: bool,
) -> None:
    """Log RAG invocation to rag_logs table. Non-blocking — failures are logged as warnings."""
    if not session_id:
        return
    import json
    try:
        with db_pool.connection() as conn:
            with conn.cursor() as cur:
                # Get the latest chat_message_id for this session (the user message that triggered this RAG call)
                cur.execute(
                    """SELECT id FROM chat_messages
                       WHERE chat_session_id = %s::uuid AND role = 'user'
                       ORDER BY created_at DESC LIMIT 1""",
                    (session_id,),
                )
                row = cur.fetchone()
                if not row:
                    return
                chat_message_id = row[0]
                cur.execute(
                    """INSERT INTO rag_logs (chat_message_id, query, chunks_retrieved, threshold_met)
                       VALUES (%s, %s, %s::jsonb, %s)""",
                    (chat_message_id, query, json.dumps(chunks_list), threshold_met),
                )
            conn.commit()
    except Exception as exc:
        import logging
        logging.getLogger(__name__).warning("Failed to log RAG invocation: %s", exc)
```

Update `create_rag_tool` signature to `create_rag_tool(db_pool, embeddings, similarity_threshold=0.45, session_id=None)` and call `_log_rag_invocation` after the query fetches rows.
