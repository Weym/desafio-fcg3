Create `mobile/lib/shared/widgets/animated_entrance.dart` with the `AnimatedEntrance` StatelessWidget.

Key requirements:
1. Uses `TweenAnimationBuilder<double>` (tween 0.0→1.0) matching the reference in `design ideas/animations.md`
2. Applies `Opacity` (value) + `Transform.translate` (Offset(0, slideOffset * (1 - value))) to child
3. Accepts parameters: `child` (required), `delay` (required Duration), `duration` (optional, defaults to `AppAnimations.entranceDuration`), `curve` (optional, defaults to `AppAnimations.entranceCurve`), `slideOffset` (optional, defaults to `AppAnimations.entranceSlideOffset`)
4. **Reduced-motion support:** Check `MediaQuery.of(context).disableAnimations` — if true, render child directly without any animation wrapper (skip TweenAnimationBuilder entirely)
5. The `delay` parameter controls stagger timing. Since `TweenAnimationBuilder` starts automatically, use a `Future.delayed` approach inside a StatefulWidget variant OR use a simpler approach: use `TweenAnimationBuilder` and ignore delay in the tween (delay is handled by the caller placing items at staggered offsets). Actually, to properly support delay: make it a `StatefulWidget` that waits `delay` duration before triggering the animation via a boolean flag + `AnimatedOpacity`/`AnimatedSlide` OR use `TweenAnimationBuilder` wrapped in a `FutureBuilder`-like approach.

**Best approach for delay:** Convert to `StatefulWidget`. In `initState`, set `_animate = false`, then `Future.delayed(delay, () { if (mounted) setState(() => _animate = true); })`. Use `AnimatedOpacity` + `AnimatedSlide` (or manual `Transform.translate` + `Opacity` with `AnimatedContainer`-like approach). Actually simplest: use the `TweenAnimationBuilder` approach but make the widget stateful to delay when the tween starts building.

**Concrete implementation:**

```dart
import 'package:flutter/material.dart';
import '../../core/theme/app_animations.dart';

/// Reusable entrance animation: fade + slide-up with configurable delay.
/// Respects reduced-motion accessibility setting.
class AnimatedEntrance extends StatefulWidget {
  final Widget child;
  final Duration delay;
  final Duration? duration;
  final Curve? curve;
  final double? slideOffset;

  const AnimatedEntrance({
    super.key,
    required this.child,
    this.delay = Duration.zero,
    this.duration,
    this.curve,
    this.slideOffset,
  });

  @override
  State<AnimatedEntrance> createState() => _AnimatedEntranceState();
}

class _AnimatedEntranceState extends State<AnimatedEntrance> {
  bool _visible = false;

  @override
  void initState() {
    super.initState();
    if (widget.delay == Duration.zero) {
      _visible = true;
    } else {
      Future.delayed(widget.delay, () {
        if (mounted) setState(() => _visible = true);
      });
    }
  }

  @override
  Widget build(BuildContext context) {
    // Respect reduced-motion accessibility setting
    final reduceMotion = MediaQuery.of(context).disableAnimations;
    if (reduceMotion) {
      return widget.child;
    }

    final duration = widget.duration ?? AppAnimations.entranceDuration;
    final curve = widget.curve ?? AppAnimations.entranceCurve;
    final offset = widget.slideOffset ?? AppAnimations.entranceSlideOffset;

    return AnimatedOpacity(
      opacity: _visible ? 1.0 : 0.0,
      duration: duration,
      curve: curve,
      child: AnimatedSlide(
        offset: _visible ? Offset.zero : Offset(0, offset / 100),
        duration: duration,
        curve: curve,
        child: widget.child,
      ),
    );
  }
}
```

Note: `AnimatedSlide` uses fractional offsets (fraction of widget size), so `offset / 100` approximates pixel-based offsets for typical widget sizes. If `AnimatedSlide` doesn't feel right, use `TweenAnimationBuilder` with a manual `_visible` trigger instead:

```dart
// Alternative if AnimatedSlide offset feels wrong:
return TweenAnimationBuilder<double>(
  tween: Tween(begin: 0.0, end: _visible ? 1.0 : 0.0),
  duration: duration,
  curve: curve,
  builder: (context, value, child) {
    return Opacity(
      opacity: value,
      child: Transform.translate(
        offset: Offset(0, offset * (1 - value)),
        child: child,
      ),
    );
  },
  child: widget.child,
);
```

Choose whichever approach produces the smoothest visual result. The `TweenAnimationBuilder` approach matches the reference code more closely. Test both — prefer the TweenAnimationBuilder approach as it matches `design ideas/animations.md` exactly.
