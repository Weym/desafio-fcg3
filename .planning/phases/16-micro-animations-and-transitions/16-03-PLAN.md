```dart
class AppAnimations {
  static const Duration staggerDelay = Duration(milliseconds: 150);
  static const int maxStaggerIndex = 5;
  static const Duration entranceDuration = Duration(milliseconds: 800);
  static const Curve entranceCurve = Curves.easeOutCubic;
  static const double entranceSlideOffset = 20.0;

  /// Centralized stagger formula. Capped at maxStaggerIndex.
  static Duration getEntranceDelay(int index) {
    return Duration(
      milliseconds: min(index, maxStaggerIndex) * staggerDelay.inMilliseconds,
    );
  }
}
```

### client_documents_screen.dart, client_chat_screen.dart, client_notifications_screen.dart, client_resources_screen.dart

**CRITICAL: Virtualization-safe pattern (Review H2 fix)**

For each of these screens, READ THE FILE FIRST to determine the list widget type:

**If using `ListView.builder` or `GridView.builder` (virtualized):**
Items are built on demand as user scrolls. Index-based stagger would cause items scrolled into view later to animate with wrong delays or re-animate. To handle this:

- Wrap ONLY the first render. Since `AnimatedEntrance` is a `StatefulWidget` that animates once on mount (the `_visible` flag persists), wrapping `ListView.builder` items with `AnimatedEntrance` is safe — each item animates once when first built. The `getEntranceDelay(index)` cap at 5 means items at index >= 5 all get the same delay (750ms), which is acceptable.
- This works because `AnimatedEntrance` animates on `initState` only (via Timer), and `ListView.builder` creates new State objects for items entering the viewport. Items scrolled back into view re-create State and re-animate, which is acceptable UX.

Pattern:
```dart
itemBuilder: (context, index) {
  return AnimatedEntrance(
    delay: AppAnimations.getEntranceDelay(index),
    child: /* existing item widget */,
  );
},
```

**If using mapped `Column` or non-virtualized `ListView` (children: [...]):**
Simpler — just wrap each child:
```dart
children: items.asMap().entries.map((entry) {
  return AnimatedEntrance(
    delay: AppAnimations.getEntranceDelay(entry.key),
    child: /* existing item widget */,
  );
}).toList(),
```

**IMPORTANT — DO NOT wrap these elements:**
- AppBar/Scaffold (structural)
- Loading skeletons/shimmer (appear before data loads)
- Error states (should appear instantly)
- Filter chips/tabs/segmented controls (interaction controls)
- Empty states (should appear instantly)
- LinearProgressIndicator or loading indicators
- SizedBox spacers between items
