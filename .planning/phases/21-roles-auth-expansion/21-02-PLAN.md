<threat_model>

## Trust Boundaries

| Boundary | Description |
| --- | --- |
| Provider JWT → Staff CRUD | Only provider role can create/modify/delete staff records |
| Self-operation guard | Provider cannot deactivate or edit their own record |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
| --- | --- | --- | --- | --- |
| T-21-06 | Elevation of Privilege | POST /staff/members | mitigate | StaffCreate schema restricts role to staff/coordinator/secretary via regex — cannot create providers |
| T-21-07 | Tampering | PUT /staff/members/{id} | mitigate | StaffUpdate.role regex prevents escalation to 'provider'; self-edit blocked by ID check |
| T-21-08 | Denial of Service | Provider self-deactivation | mitigate | Explicit ID comparison blocks DELETE on own record — prevents system lockout |
| T-21-09 | Information Disclosure | GET /staff/members | mitigate | Provider record filtered from list results (WHERE role != 'provider') |
| T-21-10 | Spoofing | Email uniqueness | mitigate | ConflictException on duplicate email prevents two staff members sharing credentials |

</threat_model>
