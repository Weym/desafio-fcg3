def check_ownership(resource_student_id: UUID, user: UserContext) -> None:

| Login flow → JWT issuance | staff.role column drives JWT role — corruption = privilege escalation |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
| --- | --- | --- | --- | --- |
| T-21-01 | Elevation of Privilege | require_staff() expansion | mitigate | Only add 'provider' to the accepted roles set — don't use wildcard or remove check |
| T-21-02 | Spoofing | Login role mapping | mitigate | Map role from DB record, not from client input; JWT role derived server-side only |
| T-21-03 | Tampering | staff.role DB value | accept | Only provider (via seed) or future staff CRUD can write to staff.role; no self-service |
| T-21-04 | Elevation of Privilege | require_provider() | mitigate | Strict equality check `user.role != "provider"` — no list, no fallback |
| T-21-05 | Denial of Service | Inactive staff login | mitigate | Return 401 early for inactive status — prevents token issuance for deactivated accounts |
