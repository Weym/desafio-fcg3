## Key Decisions from CONTEXT.md
- **D-05:** Docker Compose uses volume mounts for source code with hot-reload. All three Python services run with `uvicorn --reload`.
- **D-06:** Each service has its own `requirements.txt` in its directory.
- **D-07:** One Dockerfile per service directory. Each service builds independently.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Host → Docker containers | Docker networking exposes ports to localhost |
| Container → PostgreSQL | Database credentials in environment variables |
| .env file → Git | Secrets must not be committed |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | I (Information Disclosure) | .env file | mitigate | `.env` added to `.gitignore`; `.env.example` has placeholder values only; never real secrets |
| T-01-02 | I (Information Disclosure) | PostgreSQL port exposure | accept | Port 5432 exposed to localhost only for dev convenience; production uses Docker network isolation |
| T-01-03 | S (Spoofing) | MCP_SERVICE_TOKEN | mitigate | Token in `.env` with placeholder value; actual token must be generated per environment |
| T-01-04 | T (Tampering) | Docker image base | mitigate | Pin to `python:3.12-slim` (official); base image trust is transitive |
| T-01-05 | D (Denial of Service) | Healthcheck storm | mitigate | interval 10-15s, timeout 5s, retries 3, start_period 30s prevents rapid healthcheck polling on slow starts |
| T-01-06 | I (Information Disclosure) | .dockerignore | mitigate | `.dockerignore` excludes `.env`, `.planning/`, `scripts/seed.py` from image builds |
</threat_model>
