  <action>
    Create `backend/src/infrastructure/config.py` with Pydantic BaseSettings providing validated, type-safe, lazy configuration.

    **CRITICAL: Do NOT eagerly instantiate Settings at module load.** Use `@lru_cache` on `get_settings()` so the first caller triggers validation. This prevents crashes during Phase 1 when env vars may be incomplete.

    ```python
    from functools import lru_cache
    from typing import Literal
    from pydantic import PostgresDsn, Field
    from pydantic_settings import BaseSettings, SettingsConfigDict

    class Settings(BaseSettings):
        model_config = SettingsConfigDict(
            env_file=".env",
            env_file_encoding="utf-8",
            case_sensitive=False,
            extra="ignore",
        )

        # --- Database ---
        # FastAPI + MCP use asyncpg; LangChain uses psycopg (via DATABASE_URL_AI)
        database_url: PostgresDsn = Field(
            description="PostgreSQL connection URL (asyncpg for FastAPI runtime)"
        )

        # --- JWT ---
        jwt_secret: str = Field(
            min_length=16,
            description="Secret key for JWT signing (min 16 chars)",
        )
        jwt_algorithm: str = Field(
            default="HS256",
            description="JWT algorithm",
        )
        jwt_expiration_hours: int = Field(
            default=24, ge=1, le=168,
            description="JWT expiration in hours (1-168)",
        )

        # --- MCP / Internal ---
        mcp_service_token: str = Field(
            min_length=16,
            description="Internal service token for MCP→API authentication",
        )
        fastapi_url: str = Field(
            default="http://fastapi-app:8000",
            description="Internal FastAPI service URL",
        )

        # --- WhatsApp ---
        whatsapp_token: str = Field(
            description="WhatsApp Business Cloud API token",
        )
        whatsapp_phone_number_id: str = Field(
            description="WhatsApp Business Phone Number ID",
        )
        whatsapp_webhook_verify_token: str = Field(
            description="Custom verify token for webhook challenge",
        )
        whatsapp_app_secret: str = Field(
            default="",
            description="WhatsApp App Secret for webhook validation",
        )

        # --- Email ---
        resend_api_key: str = Field(
            pattern=r"^re_",
            description="Resend API key (starts with 're_')",
        )

        # --- LLM ---
        llm_provider: Literal["openai", "gemini"] = Field(
            default="openai",
            description="LLM provider selection",
        )
        openai_api_key: str | None = Field(
            default=None,
            description="OpenAI API key",
        )
        gemini_api_key: str | None = Field(
            default=None,
            description="Google Gemini API key",
        )

        # --- Optional ---
        fcm_credentials_path: str | None = Field(
            default=None,
            description="Path to Firebase service account JSON",
        )

    @lru_cache
    def get_settings() -> Settings:
        """Lazy singleton — first call validates and caches settings."""
        return Settings()
    ```

    **Why lazy?**
    - In Phase 1, stubs and minimal app start without all env vars. `get_settings()` is not called at module import.
    - In Phase 2+, route handlers and services call `settings = get_settings()` at runtime. If misconfigured, the error occurs on first request (clear stack trace) rather than on import (cryptic early crash).

    **Do NOT add at module level:**
    ```python
    # WRONG: This crashes on import if env vars are missing
    settings = get_settings()
    ```
