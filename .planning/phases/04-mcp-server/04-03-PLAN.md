Tool 1 — `create_enrollment` (student_id INJECTED):
```python
@mcp.tool(
    name="create_enrollment",
    description="Cria uma matricula (rascunho) com as disciplinas selecionadas. O aluno deve confirmar com confirm_enrollment depois."
)
async def create_enrollment(
    enrollment_period_id: str,
    course_ids: list[str],
    student_id: str = Depends(resolve_student_id),
    ctx: Context = CurrentContext()
) -> dict:
    client = ctx.lifespan_context["http_client"]
    data, _ = await call_api(client, "POST", "/enrollments", json={
        "student_id": student_id,
        "enrollment_period_id": enrollment_period_id,
        "course_ids": course_ids
    })
    return data
```

Note: confirm, drop, lock do NOT need `Depends(resolve_student_id)` — they operate on specific resource IDs returned by previous calls. The FastAPI side still validates ownership via the service token path.

| Boundary | Description |
|----------|-------------|
| LLM → mutation tools | Agent provides enrollment/document/appointment params; higher risk than reads |
| Tool → FastAPI | Mutations proxied with X-Service-Token; FastAPI validates business rules (prereqs, period, ownership) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-04-10 | Tampering | enrollment_id in confirm/drop/lock | mitigate | FastAPI validates ownership via dual-auth dependency (Phase 3 D-05 defense in depth); MCP does not bypass |
| T-04-11 | Repudiation | Mutation tool calls | mitigate | All calls logged to mcp_action_logs with tool_name, input_params, result, timestamp |
| T-04-12 | Elevation of Privilege | Agent calling create_enrollment for wrong student | mitigate | student_id injected by resolve_student_id from session, not from agent input |

</threat_model>
