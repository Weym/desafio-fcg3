<threat_model>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| AI service → MCP headers | `X-Chat-Session-ID` arrives from an external caller and cannot be trusted without validation |
| MCP middleware → PostgreSQL audit log | Audit writes cross into the database and are required for non-repudiation |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-06-01 | S | `mcp_server/middleware.py` | mitigate | Validate `X-Chat-Session-ID` and active session context before allowing tool execution |
| T-04-06-02 | R | `mcp_server/middleware.py` | mitigate | Remove early returns and swallowed insert exceptions so every valid call produces an auditable DB write or fails loudly |
| T-04-06-03 | D | `mcp_server/middleware.py` | mitigate | Fail closed when audit infrastructure (`db_pool`, valid session) is missing so operators see the outage instead of losing logs silently |

</threat_model>
