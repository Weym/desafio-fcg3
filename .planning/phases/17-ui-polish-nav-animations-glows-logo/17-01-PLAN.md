<objective>
Fix the bottom navigation bar animations that aren't functioning correctly after Phase 16, and add the missing 6th Support tab to client navigation.

Purpose: The GlassBottomNav is currently a StatelessWidget using AnimatedContainer + TweenAnimationBuilder. GoRouter rebuilds the widget tree on navigation, which may reconstruct (not update) these implicit animation widgets, causing the springy easeOutBack glow/scale transitions to never visually trigger. Converting to StatefulWidget with an explicit AnimationController guarantees animation state persists across rebuilds. Also, ClientShell has 6 routes (index 0-5 including Support) but only 5 NavItems, so tapping Support crashes or doesn't navigate.

Output: Working bottom nav animations on both client and staff shells, 6-tab client navigation.
</objective>

  <action>
Rewrite `GlassBottomNav` from `StatelessWidget` to `StatefulWidget` with `TickerProviderStateMixin` to fix animations not triggering on GoRouter navigation rebuilds. The core problem: `AnimatedContainer` and `TweenAnimationBuilder` are implicit animations that depend on widget identity staying the same across rebuilds — GoRouter may reconstruct the widget, causing the animation to jump to end state immediately.

**Implementation (per D-01):**

1. Change class signature:
   ```dart
   class GlassBottomNav extends StatefulWidget {
     final int currentIndex;
     final List<NavItem> destinations;
     final ValueChanged<int> onTap;
     const GlassBottomNav({super.key, required this.currentIndex, required this.destinations, required this.onTap});
     @override
     State<GlassBottomNav> createState() => _GlassBottomNavState();
   }
   ```

2. Create `_GlassBottomNavState with TickerProviderStateMixin`:
   - Create a single `AnimationController _controller` in `initState()`:
     - `duration: AppAnimations.navTransitionDuration` (300ms)
     - `vsync: this`
   - Track `_previousIndex` as an `int` field initialized to `widget.currentIndex`
   - In `didUpdateWidget()`: if `oldWidget.currentIndex != widget.currentIndex`, set `_previousIndex = oldWidget.currentIndex` and run `_controller.forward(from: 0.0)` to drive the animation from 0→1
   - Create a `CurvedAnimation _curvedAnimation` using `AppAnimations.navTransitionCurve` (easeOutBack)
   - Dispose `_controller` in `dispose()`

3. Replace the per-item `AnimatedContainer` + `TweenAnimationBuilder` approach with an `AnimatedBuilder` (listening to `_curvedAnimation`) wrapping the Row's children. For each nav item at `index`:
   - **Icon size**: If `index == widget.currentIndex`, lerp from `AppAnimations.navIconSizeDefault` to `AppAnimations.navIconSizeSelected` using `_curvedAnimation.value`. If `index == _previousIndex`, lerp from `AppAnimations.navIconSizeSelected` to `AppAnimations.navIconSizeDefault`. Otherwise, use `AppAnimations.navIconSizeDefault`.
   - **Glow box shadow**: If `index == widget.currentIndex`, lerp glow alpha from `0.0` to `0.35`, blurRadius from `0` to `AppAnimations.navGlowBlurSelected` (16), spreadRadius from `0` to `AppAnimations.navGlowSpreadSelected` (4) — all using `_curvedAnimation.value`. If `index == _previousIndex`, do the reverse (fade out). Otherwise, no glow.
   - **Background color**: If selected, lerp alpha from `0.0` to `0.2` on `AppColors.neonTeal` using `_curvedAnimation.value`.
   - **Icon color**: Selected → `AppColors.neonTeal`, unselected → `colors.onSurfaceVariant` (no lerp needed for color, instant switch is fine).
   - **Label color**: Same as icon color — selected `AppColors.neonTeal`, unselected `colors.onSurfaceVariant`.

4. **Reduced-motion (per existing pattern):** Check `MediaQuery.of(context).disableAnimations`. If true, set `_controller.value = 1.0` directly (skip animation, snap to final state).

5. Keep the outer structure identical: `ClipRRect` → `BackdropFilter(blur: 20)` → `Container(height: 80+bottomPadding)` → `Row(spaceAround)` → items. The visual output must be identical to current design, only the animation mechanism changes.

6. **Per D-02:** The animation parameters (curve `easeOutBack`, duration 300ms, icon sizes 24→28, glow spread 4/blur 16) remain as constants in `AppAnimations`. The agent may tweak the lerp interpolation (e.g., add slight overshoot on glow) if it improves the springy feel, but the constants file is the source of truth.

**Keep:** `NavItem` data class unchanged (exported, used by shells). `GlassBottomNav` constructor signature unchanged (same `currentIndex`, `destinations`, `onTap` params) — zero breaking changes for consumers.
