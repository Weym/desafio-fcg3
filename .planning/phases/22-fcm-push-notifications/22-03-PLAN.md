1. **Documents controller** (`backend/src/features/documents/controllers.py`):
   - Find the endpoint that calls `document_service.update_document_status()` (staff-only status update endpoint).
   - AFTER the service call returns successfully AND the new status is "ready", dispatch notification:
     ```python
     import asyncio
     from src.features.notifications.services import notification_service
     from src.infrastructure.database import get_db_session as get_fresh_session

     # After update_document_status returns:
     if data.status == "ready":
         async def _send_notification():
             async for fresh_db in get_fresh_session():
                 try:
                     await notification_service.notify_document_ready(
                         fresh_db, document.student_id, document.type, document.id
                     )
                 finally:
                     await fresh_db.close()
         asyncio.create_task(_send_notification())
     ```
   - Per D-11: use `asyncio.create_task()` to make it non-blocking. The notification uses a fresh DB session because the original request session may close before the background task completes.

2. **Enrollment controller** (`backend/src/features/enrollment/controllers.py`):
   - Find the `confirm_enrollment` endpoint handler.
   - AFTER the service call returns successfully, dispatch notification:
     ```python
     import asyncio
     from src.features.notifications.services import notification_service

     # After confirm_enrollment returns:
     async def _send_notification():
         async for fresh_db in get_fresh_session():
             try:
                 await notification_service.notify_enrollment_confirmed(
                     fresh_db, user.id, enrollment_id
                 )
             finally:
                 await fresh_db.close()
     asyncio.create_task(_send_notification())
     ```
   - Note: enrollment confirmation is done BY the student (confirms their own draft). The student receives push confirmation that their action succeeded.

3. **Appointments controller** (`backend/src/features/appointments/controllers.py`):
   - There is NO explicit "confirm appointment" endpoint currently. Appointments go from `scheduled` directly. Per the ROADMAP requirement FCM-05 ("appointment confirmed"), the trigger should fire when an appointment is BOOKED (status=scheduled) since that IS the confirmation in the current model.
   - Find the `book_appointment` endpoint (POST /appointments).
   - AFTER `appointment_service.book_appointment()` returns:
     ```python
     import asyncio
     from src.features.notifications.services import notification_service

     async def _send_notification():
         async for fresh_db in get_fresh_session():
             try:
                 await notification_service.notify_appointment_confirmed(
                     fresh_db, user.id, response.id
                 )
             finally:
                 await fresh_db.close()
     asyncio.create_task(_send_notification())
     ```
   - This notifies the student that their appointment booking was successfully confirmed/created.

Important pattern: Each `_send_notification` closure acquires its own DB session via `get_fresh_session()` (the existing `get_db_session` generator from `src.infrastructure.database`). This prevents "Session is closed" errors when the original request session is committed and closed before the background task runs. Always close the session in a finally block.
