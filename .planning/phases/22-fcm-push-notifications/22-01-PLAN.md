1. Create `backend/src/features/notifications/services.py` with class `NotificationService`:

   - Module-level Firebase init: At import time or via an `init_firebase()` function called from `main.py` lifespan:
     ```python
     import firebase_admin
     from firebase_admin import credentials, messaging
     from src.infrastructure.config import settings

     def init_firebase() -> None:
         if settings.fcm_credentials_path and not firebase_admin._apps:
             cred = credentials.Certificate(settings.fcm_credentials_path)
             firebase_admin.initialize_app(cred)
     ```
     If `fcm_credentials_path` is None, log a warning and make all send operations no-op (graceful degradation per D-12 fire-and-forget).

   - `async def send_push(self, db: AsyncSession, student_id: UUID, event: NotificationEvent, title: str, body: str, data: dict[str, str] | None = None) -> None`:
     - Query all FcmToken records for `student_id`
     - For each token, build `messaging.Message(notification=messaging.Notification(title=title, body=body), data=data or {}, token=token_str)` (per D-08: notification + data fields)
     - Call `messaging.send(message)` (synchronous — wrap in `asyncio.to_thread` or use directly since firebase-admin is blocking)
     - On `messaging.UnregisteredError` or `messaging.SenderIdMismatchError`: delete that FcmToken from DB (per D-05)
     - On any other exception: log error, continue to next token (per D-12 fire-and-forget, no retry)
     - All notification content in Portuguese (per specifics in CONTEXT.md)

   - Helper methods for each event type (called by feature services):
     - `async def notify_document_ready(self, db: AsyncSession, student_id: UUID, document_type: str, document_id: UUID) -> None`
       Calls `send_push` with title="Documento pronto", body=f"Seu {document_type} está pronto para retirada", data={"event": "document_ready", "document_id": str(document_id)} (per D-09 personalized)

       Calls `send_push` with title="Agendamento confirmado", body="Seu agendamento foi confirmado", data={"event": "appointment_confirmed", "appointment_id": str(appointment_id)}

   - Module-level singleton: `notification_service = NotificationService()`

<threat_model>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Mobile app → FCM token endpoint | Untrusted client sends token string |
| Backend → Firebase API | Internal service calls Firebase with credentials |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-22-01 | Spoofing | PUT /fcm-token | mitigate | IDOR check: student can only register tokens for own student_id. Service token for internal calls. |
| T-22-02 | Tampering | FCM token string | accept | Firebase validates token on send; invalid tokens cleaned up per D-05. No sensitive data in token itself. |
| T-22-03 | Information Disclosure | Notification payload | mitigate | Data field contains only IDs (UUIDs), not PII. Notification body is generic enough ("Documento pronto: {type}"). |
| T-22-04 | Denial of Service | Token registration | mitigate | Unique constraint prevents duplicate tokens. Rate limiting via existing slowapi middleware. |
| T-22-05 | Elevation of Privilege | Service token abuse | mitigate | FCM_CREDENTIALS_PATH only accessible inside container. Service token validated by existing middleware. |

</threat_model>
