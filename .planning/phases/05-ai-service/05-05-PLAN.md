**Request/response schema:**

```python
from pydantic import BaseModel

class ChatRequest(BaseModel):
    session_id: str  # UUID of the chat session
    message: str     # Student's text message

class ChatResponse(BaseModel):
    response: str    # Agent's response text
    session_id: str  # Echo back for correlation
```

**Endpoint implementation:**

```python
from ai_service.agent import invoke_agent
from ai_service.database import save_chat_message
from ai_service.config import settings

@app.post("/chat")
async def chat(request: ChatRequest):
    """
    Process a student message through the AI agent.

    Per D-04: Trusts the caller (FastAPI backend) for session validity.
    Per D-09: Writes the assistant response to chat_messages after generation.

    Called by FastAPI backend during webhook processing (Phase 6).
    Can also be called directly for testing without WhatsApp.
    """
    try:
        # 1. Invoke the agent (D-08: loads history, runs agent, returns response)
        response_text = await invoke_agent(
            settings=settings,
            db_pool=app.state.db_pool,
            system_prompt=app.state.system_prompt,
            session_id=request.session_id,
            user_message=request.message,
        )

        # 2. Save assistant response to chat_messages (D-09)
        save_chat_message(
            pool=app.state.db_pool,
            session_id=request.session_id,
            role="assistant",
            content=response_text,
        )

        # 3. Return the response
        return ChatResponse(
            response=response_text,
            session_id=request.session_id,
        )

    except Exception as e:
        import logging
        logging.getLogger(__name__).error(f"Chat error for session {request.session_id}: {e}")
        # Return a fallback response rather than a 500 error
        # The agent already has its own fallback — this catches unexpected errors
        fallback = "Desculpe, estou com dificuldades tecnicas. Tente novamente em alguns minutos."
        # Still save the fallback as the assistant response
        try:
            save_chat_message(
                pool=app.state.db_pool,
                session_id=request.session_id,
                role="assistant",
                content=fallback,
            )
        except Exception:
            pass  # Don't fail if save fails
        return ChatResponse(
            response=fallback,
            session_id=request.session_id,
        )
```

**Key decisions:**
- Per D-04: No session validation in the AI service. The caller (FastAPI backend) validates the session before calling.
- Per D-09: The AI service writes the assistant response to `chat_messages` table. The service owns the full lifecycle: agent invocation → save response → return to caller.
- Error handling: Even on failure, return a valid ChatResponse with a fallback message. Never return a raw 500 error to the webhook flow — that would leave the student without a response.
- The endpoint is async (`async def`) since `invoke_agent` is async (awaits MCP tool calls).
