**Part A — api_client.py: Add student_id parameter to call_api_raw and call_api**

The fix is centralized: both `call_api_raw` and `call_api` gain a `student_id: str | None = None` keyword parameter. When provided, the function merges `{"X-Student-Id": student_id}` into the request headers before passing to `client.request()`.

1. In `call_api_raw`, change the signature from `**kwargs: Any` to an explicit `student_id: str | None = None` parameter BEFORE `**kwargs`:

```python
async def call_api_raw(
    client: httpx.AsyncClient,
    method: str,
    path: str,
    *,
    student_id: str | None = None,
    **kwargs: Any,
) -> tuple[httpx.Response, bool]:
```

2. At the TOP of `call_api_raw` (before the first try block), add header merging logic:

```python
    if student_id is not None:
        existing_headers = kwargs.get("headers") or {}
```

This ensures both the initial request AND the retry send the same headers (kwargs is reused for both `client.request()` calls).

```python
async def call_api(
```

**Part C — Add header injection regression tests to test_api_client.py**

Add these tests to `mcp_server/tests/test_api_client.py`:

```python
async def test_call_api_injects_x_student_id_header(mock_http_client):
    mock_http_client.request.return_value = _make_response(200, {"ok": True})

    data, retried = await call_api(

    assert retried is False
    call_kwargs = mock_http_client.request.call_args
    assert call_kwargs.kwargs.get("headers", {}).get("X-Student-Id") == "abc" or \

async def test_call_api_omits_header_when_student_id_is_none(mock_http_client):
    mock_http_client.request.return_value = _make_response(200, {"ok": True})

    data, _ = await call_api(mock_http_client, "GET", "/health")

    call_kwargs = mock_http_client.request.call_args
    headers = call_kwargs.kwargs.get("headers") or call_kwargs[1].get("headers") or {}
    assert "X-Student-Id" not in headers


async def test_call_api_preserves_existing_headers_when_injecting_student_id(mock_http_client):
    mock_http_client.request.return_value = _make_response(200, {"ok": True})

    data, _ = await call_api(
        mock_http_client, "GET", "/resource",
        student_id="stu-1",
        headers={"X-Custom": "value"},
    )

    call_kwargs = mock_http_client.request.call_args
    headers = call_kwargs.kwargs.get("headers") or call_kwargs[1].get("headers") or {}
    assert headers.get("X-Student-Id") == "stu-1"
    assert headers.get("X-Custom") == "value"


async def test_call_api_sends_student_id_header_on_retry(mock_http_client):
    """X-Student-Id must be present on both the initial request and the retry."""
    mock_http_client.request.side_effect = [
        _make_response(500, {"error": {"code": "SERVER_ERROR"}}),
        _make_response(200, {"ok": True}),
    ]

    data, retried = await call_api(
        mock_http_client, "GET", "/grades", student_id="stu-retry"
    )

    assert retried is True
    # Both calls should have the header
    for call_obj in mock_http_client.request.call_args_list:
        headers = call_obj.kwargs.get("headers") or call_obj[1].get("headers") or {}
        assert headers.get("X-Student-Id") == "stu-retry"
```

Note on assertion style: `mock_http_client.request` is an `AsyncMock`. When `call_api_raw` calls `await client.request(method, path, **kwargs)`, httpx passes positional + keyword args. Access the headers from `call_args`. The exact attribute path depends on how the mock records it — use `call_args.kwargs` first, fallback to `call_args[1]` (the second positional tuple element for keyword-style access).

A simpler and more reliable assertion pattern:

```python
# Get the kwargs dict that was passed to client.request()
_, called_kwargs = mock_http_client.request.call_args
# OR if call_args uses named tuple:
called_kwargs = mock_http_client.request.call_args.kwargs
```

The executor should verify which access pattern works by checking the first existing test (`test_call_api_returns_payload_on_success`) — the mock_http_client in conftest.py is just `AsyncMock()` so `call_args` will be a `call` object. Use whichever pattern is consistent.
