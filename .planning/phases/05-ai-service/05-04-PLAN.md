Create `ai_service/mcp_tools.py` — per D-01/D-02/D-03.

**Design:** A function that loads MCP tools from the MCP server with a per-request `X-Chat-Session-ID` header.

```python
from langchain_mcp_adapters.client import MultiServerMCPClient

async def load_mcp_tools(mcp_server_url: str, session_id: str) -> list:
    """
    Load MCP tools from the MCP server with session-specific headers.

    Per D-01: Uses langchain-mcp-adapters for auto-discovery.
    Per D-03: X-Chat-Session-ID header passed per request.

    Args:
        mcp_server_url: MCP server streamable-http URL (e.g., "http://mcp-server:8002/mcp")
        session_id: Chat session UUID for student_id resolution on MCP side

    Returns:
        List of LangChain-compatible tools
    """
    client = MultiServerMCPClient(
        {
            "academic-mcp": {
                "transport": "http",
                "url": mcp_server_url,
                "headers": {
                    "X-Chat-Session-ID": session_id,
                },
            }
        }
    )
    tools = await client.get_tools()
    return tools
```

**Key points:**
- Per D-02: Tool discovery is done per request (MultiServerMCPClient is lightweight). The tools themselves are auto-discovered from the MCP server's registered tools.
- Per D-03: `X-Chat-Session-ID` is the only per-request metadata. MCP resolves `student_id` from this header.
- The function is `async` because `MultiServerMCPClient.get_tools()` is async.
- Each invocation creates a new client with the specific session_id — this is necessary because headers are set at client construction time.

**Design:** Two main functions:

**Implementation:**

```python
import asyncio
from langchain.agents import create_agent
from langchain_core.messages import HumanMessage, SystemMessage

from ai_service.llm_factory import get_model_string
from ai_service.mcp_tools import load_mcp_tools
from ai_service.rag import create_rag_tool
from ai_service.database import load_chat_history

FALLBACK_MESSAGE = "Desculpe, nao consegui processar sua solicitacao no momento. Tente novamente ou entre em contato com a secretaria."

async def invoke_agent(
    settings,
    db_pool,
    system_prompt: str,
    session_id: str,
    user_message: str,
) -> str:
    """
    Process a student message through the ReAct agent.

    Per D-08: Stateless per request — loads last 20 messages from DB.
    Per D-07: max_iterations=10, max_execution_time=45s.

    Args:
        settings: Application settings
        db_pool: psycopg3 connection pool
        system_prompt: Loaded system prompt text
        session_id: Chat session UUID
        user_message: Student's text message

    Returns:
        Agent's response text
    """
    # 1. Load MCP tools with session-specific header (D-01/D-03)
    mcp_tools = await load_mcp_tools(settings.MCP_SERVER_URL, session_id)

    # 2. Create RAG tool (D-11)
    rag_tool = create_rag_tool(db_pool, settings.OPENAI_API_KEY)

    # 3. Combine all tools
    all_tools = mcp_tools + [rag_tool]

    # 4. Create the agent (D-07)
    model_string = get_model_string(settings)
    agent = create_agent(
        model=model_string,
        tools=all_tools,
        system_prompt=system_prompt,
    )

    # 5. Load conversation history (D-08: last 20 messages)
    history_messages = load_chat_history(db_pool, session_id, k=settings.CHAT_HISTORY_K)

    # 6. Build the full message list
    all_messages = history_messages + [HumanMessage(content=user_message)]

    # 7. Invoke the agent with timeout (D-07)
    try:
        result = await asyncio.wait_for(
            agent.ainvoke({"messages": all_messages}),
            timeout=settings.MAX_AGENT_EXECUTION_TIME,
        )
        # Extract the last assistant message
        response_messages = result.get("messages", [])
        if response_messages:
            last_msg = response_messages[-1]
            # Handle different response formats
            if hasattr(last_msg, 'content'):
                return last_msg.content if isinstance(last_msg.content, str) else str(last_msg.content)
        return FALLBACK_MESSAGE
    except asyncio.TimeoutError:
        return FALLBACK_MESSAGE
    except Exception as e:
        # Log the error but return a user-friendly message
        import logging
        logging.getLogger(__name__).error(f"Agent error for session {session_id}: {e}")
        return FALLBACK_MESSAGE
```

**Key decisions:**
- Per D-07: `max_execution_time=45s` enforced via `asyncio.wait_for`. The `create_agent` may also support `recursion_limit` for max iterations — set appropriately if the API supports it.
- Per D-08: History loaded fresh from DB each invocation — no in-memory caching.
- Per D-23: Confirmation before mutating actions is handled by the system prompt rules, not code guards.
- Per D-06: System prompt passed as parameter (loaded from file in main.py lifespan).
- The agent is created fresh per request because tools include session-specific MCP tools. This is by design — the MCP client has session-specific headers.
- Fallback message: Generic Portuguese message when agent fails or times out. This matches the error handling table in `docs/chatbot.md` ("Desculpe, estou com dificuldades tecnicas...").
- Uses `agent.ainvoke` (async) for compatibility with the FastAPI async endpoint.
