## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| FastAPI backend → AI service `/chat` | Trusted internal caller sends untrusted message text + session_id into the AI runtime |
| AI service agent result → HTTP response | LangChain emits mixed message types that must be filtered before student-facing output |
| AI service `/chat` → `chat_messages` table | Request content crosses into durable audit/history storage |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-06-01 | I | `ai_service/agent.py` response extraction | mitigate | Walk backward to the last `AIMessage` only so tool payloads or intermediate reasoning artifacts are never returned as the student-facing answer. |
| T-05-06-02 | R | `ai_service/main.py` chat persistence | mitigate | Persist the inbound `role='user'` message before agent execution and persist the assistant reply/fallback after execution so the stored conversation can reconstruct the full exchange. |
| T-05-06-03 | D | `POST /chat` execution path | accept | Existing `MAX_AGENT_ITERATIONS` and `MAX_AGENT_EXECUTION_TIME` guards already cap runaway executions; this gap-closure plan does not widen the attack surface. |
