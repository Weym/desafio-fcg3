Create `ai_service/ingest.py` — per D-18/D-19/D-20/D-21/D-22.

**Script structure:**

1. **Configuration section:**
   - Read `DATABASE_URL` and `OPENAI_API_KEY` from environment.
   - Define `KNOWLEDGE_DIR = Path(__file__).parent / "knowledge"`.
   - Define `CATEGORY_MAP` per D-19:
     ```python
     CATEGORY_MAP = {
         'matricula.md': 'regras_matricula',
         'regulamento.pdf': 'regulamento',
         'faq.md': 'faq',
         'calendario.md': 'agendamento',
         'curriculo.md': 'curriculo',
     }
     ```

2. **Document loading function:**
   - For `.md` files: use `langchain_community.document_loaders.TextLoader` (or just `Path.read_text()`).
   - For `.pdf` files: use `langchain_community.document_loaders.PyPDFLoader` (per D-20). Wrap in try/except — if PDF parsing fails (placeholder file), log warning and skip.
   - Return raw text content per file.

3. **Chunking function:**
   - Use `RecursiveCharacterTextSplitter.from_tiktoken_encoder(chunk_size=500, chunk_overlap=50, encoding_name="cl100k_base")` per docs/chatbot.md spec (500 tokens, 50 overlap).
   - Split each document's text into chunks.
   - Return list of (chunk_text, chunk_index) tuples.

4. **Embedding function:**
   - Use `langchain_openai.OpenAIEmbeddings(model="text-embedding-3-small")` per D-16.
   - Batch embed all chunks: `embeddings.embed_documents([chunk.text for chunk in chunks])`.
   - Returns list of 1536-dimensional vectors.

5. **Database operations (using psycopg3 sync):**
   - Connect via `psycopg.connect(DATABASE_URL)`.
   - Per D-18 (delete-then-insert per source file):
     ```python
     with conn.cursor() as cur:
         # Delete existing chunks for this source
         cur.execute("DELETE FROM knowledge_base_chunks WHERE source = %s", [source_filename])
         # Insert new chunks
         for i, (content, embedding) in enumerate(zip(chunk_texts, vectors)):
             cur.execute(
                 "INSERT INTO knowledge_base_chunks (content, embedding, source, category, chunk_index) VALUES (%s, %s::vector, %s, %s, %s)",
                 [content, str(embedding), source_filename, category, i]
             )
     conn.commit()
     ```
   - Note: pgvector expects the vector as a string representation of a list: `"[0.1, 0.2, ...]"`. Use `str(vector_list)` or format appropriately.

6. **Console report (per D-21):**
   - Print: total documents processed, total chunks generated, chunks per category, total execution time.
   - Format clearly for stdout.

7. **DB audit log (per D-21):**
   - After all documents processed, insert a summary record. Use a simple table or the existing schema. Since no specific audit table exists, create a record in `knowledge_base_chunks` with `source='_ingest_log'`, `category='system'`, `content=JSON summary`, and a zero vector. OR better: just log to stdout and skip DB audit if no audit table exists — the executor can decide the simplest approach.
   - Simplest approach: print the audit summary to stdout AND write a JSON file `ai_service/knowledge/.last_ingest.json` with the summary. This avoids needing a new table.

8. **CLI entrypoint:**
   ```python
   if __name__ == "__main__":
       import argparse
       parser = argparse.ArgumentParser(description="Ingest knowledge base documents")
       parser.add_argument("--source", default=str(KNOWLEDGE_DIR), help="Path to knowledge docs")
       parser.add_argument("--chunk-size", type=int, default=500, help="Chunk size in tokens")
       parser.add_argument("--overlap", type=int, default=50, help="Chunk overlap in tokens")
       args = parser.parse_args()
       main(args.source, args.chunk_size, args.overlap)
   ```
