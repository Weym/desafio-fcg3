    **mcp_server/middleware.py** — Add `gen_random_uuid()` for the `id` column in the `_log_call` INSERT statement.

    Replace the INSERT query in `_log_call` (lines 100-112) with:
    ```python
    await db_pool.execute(
        """
        INSERT INTO mcp_action_logs (
            id,
            chat_session_id,
            tool_name,
            input_params,
            output_result,
            reasoning,
            latency_ms,
            retry,
            status
        )
        VALUES (gen_random_uuid(), $1, $2, $3::jsonb, $4::jsonb, $5, $6, $7, $8)
        """,
        chat_session_id,
        tool_name,
        _to_json_payload(input_params) or "{}",
        _to_json_payload(_serialize_result(result)),
        None,
        latency_ms,
        retry,
        effective_status,
    )
    ```

    The change adds `id` to the column list and `gen_random_uuid()` to the VALUES — this is a PostgreSQL server-side function that generates a cryptographically random UUIDv4, consistent with the pattern used for `chat_messages.id` in `ai_service/database.py`.

    The positional parameter references ($1-$8) remain unchanged because `gen_random_uuid()` is a SQL function call, not a bound parameter.

    **mcp_server/tests/test_middleware_logging.py** — Update assertions to verify the `id` column is now included in the INSERT:

    In `test_tool_logging_middleware_logs_successful_calls` (line 54), update the assertion:
    ```python
    assert "INSERT INTO mcp_action_logs" in query
    assert "gen_random_uuid()" in query  # NEW — verify UUID generation
    ```

    The 9-value positional unpacking (`query, chat_session_id, tool_name, ...`) remains unchanged because the bound parameters $1-$8 are still the same 8 Python values — `gen_random_uuid()` is computed server-side in SQL.

    Gap reason: The `mcp_action_logs.id` column is `UUID NOT NULL` with no `server_default` in the Alembic migration. The INSERT must explicitly provide the value. Without it, PostgreSQL raises "null value in column 'id' violates not-null constraint" which crashes MCP tool calls and causes cascading agent failures.
