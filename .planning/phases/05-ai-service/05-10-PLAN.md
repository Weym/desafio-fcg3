  <action>
    **Gap:** `mcp_server/middleware.py` line 100-112 INSERT INTO `mcp_action_logs` omits the `id` column. The table schema defines `id UUID NOT NULL` with no `server_default=gen_random_uuid()`, so every INSERT without an explicit `id` value crashes with: `null value in column "id" of relation "mcp_action_logs" violates not-null constraint`. This was reported in 05-UAT.md Additional Findings and blocks end-to-end agent flow.

    **mcp_server/middleware.py — Add `id` with `gen_random_uuid()` to INSERT:**

    Change the INSERT query in `_log_call` method (lines 100-112) from:

    ```python
    await db_pool.execute(
        """
        INSERT INTO mcp_action_logs (
            chat_session_id,
            tool_name,
            input_params,
            output_result,
            reasoning,
            latency_ms,
            retry,
            status
        )
        VALUES ($1, $2, $3::jsonb, $4::jsonb, $5, $6, $7, $8)
        """,
        chat_session_id,
        tool_name,
        _to_json_payload(input_params) or "{}",
        _to_json_payload(_serialize_result(result)),
        None,
        latency_ms,
        retry,
        effective_status,
    )
    ```

    To:

    ```python
    await db_pool.execute(
        """
        INSERT INTO mcp_action_logs (
            id,
            chat_session_id,
            tool_name,
            input_params,
            output_result,
            reasoning,
            latency_ms,
            retry,
            status
        )
        VALUES (gen_random_uuid(), $1, $2, $3::jsonb, $4::jsonb, $5, $6, $7, $8)
        """,
        chat_session_id,
        tool_name,
        _to_json_payload(input_params) or "{}",
        _to_json_payload(_serialize_result(result)),
        None,
        latency_ms,
        retry,
        effective_status,
    )
    ```

    Key changes:
    - Added `id,` to the column list
    - Added `gen_random_uuid(),` before `$1` in VALUES
    - The positional parameters `$1` through `$8` stay the same — `gen_random_uuid()` is a server-side function call, not a client-provided parameter

    **mcp_server/tests/test_middleware_logging.py — Update assertions:**

    The tests destructure `db_pool.execute.await_args.args` into positional variables. Since the number and order of Python parameters passed to `execute` does NOT change (we only added a server-side function call in the SQL string), the test destructuring stays the same.
