<objective>
Add configurable embedding provider (openai or openrouter) and model selection via environment variables.

Purpose: The current embedding setup is hardcoded to `OpenAIEmbeddings` with `text-embedding-3-small`. Users who use OpenRouter for LLM also want to route embeddings through OpenRouter, and may want to choose different embedding models. This mirrors the existing `LLM_PROVIDER`/`LLM_MODEL` pattern already in `llm_factory.py`.

Output: `EMBEDDING_PROVIDER` and `EMBEDDING_MODEL` env vars control embedding behavior across ingest and RAG, with backward-compatible defaults.
</objective>

@ai_service/llm_factory.py — Reference pattern for provider-agnostic factory

</context>

<task type="auto">
  <name>Task 1: Add embedding config + factory</name>
  <files>ai_service/config.py, ai_service/embedding_factory.py</files>
  <action>
**In `ai_service/config.py`:**
Add two new fields to the `Settings` dataclass:
- `EMBEDDING_PROVIDER: str = os.environ.get("EMBEDDING_PROVIDER", "openai")` — accepts `"openai"` or `"openrouter"`
- `EMBEDDING_MODEL: str = os.environ.get("EMBEDDING_MODEL", "text-embedding-3-small")` — the embedding model name

Place them after the existing `OPENROUTER_API_KEY` field, grouping them logically with the other provider settings.

**Create `ai_service/embedding_factory.py`:**
Follow the same pattern as `llm_factory.py`. Create a single function:

```python
def create_embeddings(settings: Any) -> Embeddings:
```

- If `settings.EMBEDDING_PROVIDER == "openai"`: return `OpenAIEmbeddings(model=settings.EMBEDDING_MODEL, api_key=settings.OPENAI_API_KEY)`
- If `settings.EMBEDDING_PROVIDER == "openrouter"`: return `OpenAIEmbeddings(model=settings.EMBEDDING_MODEL, api_key=settings.OPENROUTER_API_KEY, base_url="https://openrouter.ai/api/v1")`
- Otherwise: raise `ValueError` with supported providers list

Import `OpenAIEmbeddings` lazily inside each branch (same pattern as `llm_factory.py`). Type hint the return as `langchain_core.embeddings.Embeddings`.

Also add a helper function:
```python
def get_embedding_api_key(settings: Any) -> str:
```
This returns the correct API key for the configured provider (used by ingest which needs the raw key for its `IngestSettings`). Raises `RuntimeError` if the required key is missing.
